# 🕳️ Porthole

**Expose your local AI models and servers to the internet — no router config needed.**

Tunnels any local service (Ollama, Ktor, FastAPI, …) through a cheap VPS using [FRP](https://github.com/fatedier/frp).

## How it works

```
You          Your Home PC                    VPS (DigitalOcean, etc.)       Internet
──────   ──────────────────────────    ──────────────────────────────   ──────────────
         ollama  :11434               frps listens on :7000
         ktor    :8080    ──frpc──►   and exposes all tunnels       ◄── any client
         fastapi :8000                on their remote ports
```

No firewall rules on your router. The tunnel is initiated **outbound** from your machine, so it punches through NAT automatically.

## Requirements

| Where | What |
|---|---|
| Home PC | Docker + Docker Compose |
| VPS | Docker + Docker Compose, open ports 7000 / 7500 + your service ports |

## Quick start

### 1 — Generate a shared secret

```bash
openssl rand -hex 32
# e.g. a3f9c2d1e8b7…  ← use this as FRP_TOKEN everywhere
```

### 2 — Configure services

Edit `services.conf` to declare what you want to expose:

```
# name : local_port : remote_port : local_host (optional)
ollama      : 11434 : 11434 : ollama   # docker service → use its container name
ktor        : 8080  : 8080             # native process → leave blank
fastapi     : 8000  : 8000             # native process → leave blank
```

Run the generator:

```bash
./setup.sh
```

This regenerates `local/frpc.ini` and `vps/docker-compose.yml` and prints which firewall ports to open.

### 3 — Set up the VPS

```bash
scp -r vps/ root@<VPS_IP>:~/porthole/
ssh root@<VPS_IP>

cd ~/porthole/vps/
cp .env.example .env
# Fill in FRP_TOKEN and DASHBOARD_PWD

export $(cat .env | xargs)
sed -i "s/{{ FRP_TOKEN }}/$FRP_TOKEN/g"         frps.ini
sed -i "s/{{ DASHBOARD_PWD }}/$DASHBOARD_PWD/g" frps.ini

docker compose up -d
```

### 4 — Set up your local PC

```bash
cd local/
cp .env.example .env
# Set VPS_HOST and FRP_TOKEN (same token as above)

docker compose up -d

# Pull a model (first run only)
docker exec -it local-ollama-1 ollama pull llama3
```

### 5 — Test it

```bash
curl http://<VPS_IP>:11434/api/tags          # Ollama
curl http://<VPS_IP>:8080/                   # Ktor
curl http://<VPS_IP>:8000/docs               # FastAPI
```

## Adding / removing services

`services.conf` is the single source of truth. After any change, just:

```bash
./setup.sh
docker compose -f local/docker-compose.yml up -d --force-recreate frpc
# then push vps/ to your server and docker compose up -d there
```

### `local_host` field

| Service type | `local_host` to use | Example |
|---|---|---|
| In the same `local/docker-compose.yml` | container/service name | `ollama` |
| Native process on your machine | *(leave blank)* | — |

> Blank entries default to `host.docker.internal`, which resolves to your machine from inside Docker on both macOS and Linux.

## Server examples

### Ollama (Docker)

```
ollama : 11434 : 11434 : ollama
```

Pull models:
```bash
docker exec -it local-ollama-1 ollama pull llama3
docker exec -it local-ollama-1 ollama pull mistral
```

### Ktor

Bind to `0.0.0.0` in your app:

```kotlin
embeddedServer(Netty, port = 8080, host = "0.0.0.0") { ... }.start(wait = true)
```

```
# services.conf
ktor : 8080 : 8080
```

### FastAPI

```bash
uvicorn main:app --host 0.0.0.0 --port 8000
```

```
# services.conf
fastapi : 8000 : 8000
```

## DigitalOcean setup (recommended VPS)

1. **Create → Droplets → Marketplace → Docker on Ubuntu** ($6/mo, 1 GB RAM is enough)
2. Add a **Cloud Firewall** with inbound rules for TCP ports: `22`, `7000`, `7500`, plus your service ports
3. Follow [Quick start §3](#3--set-up-the-vps) above

## GPU support (NVIDIA)

Uncomment the `deploy.resources` block in `local/docker-compose.yml` and install the
[NVIDIA Container Toolkit](https://docs.nvidia.com/datacenter/cloud-native/container-toolkit/latest/install-guide.html) on your host.

## Security notes

- Tunnel connections are authenticated via `FRP_TOKEN` but traffic is **not TLS-encrypted** by default.
- For production, put **Nginx + Certbot** in front of your service ports on the VPS, or set `tls_enable = true` in `frps.ini` / `frpc.ini`.
- Never commit `.env` files — `.gitignore` already excludes them.
- Rotate `FRP_TOKEN` by updating both `.env` files and restarting both stacks.
